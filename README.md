# TODO列表 — 操作说明

一个本地离线 TODO 列表管理工具，适合刷题、学习计划等场景。基于 Electron + React + Ant Design + SQLite。

## 目录结构

应用左侧为树形目录，层级为：**文件夹** → **题单** → **Part（分区）** → **题目**
- 文件夹可无限嵌套
- 题单可直接放在根目录或文件夹内
- Part 用于给题单内的题目分组
- 题目可放在 Part 内，或直接放在题单下（无 Part 时）

## 基本操作

### 新建

| 操作 | 方式 |
|---|---|
| 新建文件夹 | 工具栏「新建文件夹」按钮 |
| 新建题单 | 工具栏「新建题单」按钮 |
| 新建 Part | 打开题单后，点击右侧「新建 Part」 |
| 新建题目 | 打开题单后，点击「新建题目」；或在 Part 内点击「+」|

新建文件夹/题单时会提示选择节点：选中文件夹则创建在其下，选中题单或未选中则创建在根目录。

### 右键菜单

在左侧树形目录中右键点击任意节点：

| 节点类型 | 菜单项 |
|---|---|
| 文件夹 | 重命名、删除 |
| 题单 | 重命名、删除、导出 Markdown |
| Part | 重命名、删除 |
| 题目 | 编辑名称、删除、复制名称 |

### 完成状态

- 点击题目前的复选框切换完成状态
- 已完成的题目显示删除线
- 各层级（Part / 题单 / 文件夹 / 全局）顶部自动统计完成率

### 排序

- 每个题目行右侧有上移/下移箭头按钮
- 题目顺序在 Part 内或题单内独立维护

### 复制题目名称

- 点击题目行右侧的复制按钮（或右键 → 复制名称）
- 名称自动复制到剪贴板

## 描述备注

文件夹和题单可以添加 Markdown 描述文本：
- 描述显示在内容区顶部
- 支持 `[链接文本](url)` Markdown 超链接语法（点击在新标签页打开）
- 点击「编辑描述」修改，点击复制图标复制全文

## 导入

### Markdown 导入

支持导入 `.md` 文件（工具栏 → 导入 → 选择文件）：

**标题层级映射：**

| Markdown | 对应 |
|---|---|
| `# 路径/题单名` | 题单（路径可选，`/a/b` 表示在 a/b 目录下） |
| `## Part名` | Part |
| `### 题目名` | 题目（可多条） |
| `- 题目名` 或 `1. 题目名` | 题目（纯文本列表） |

- 支持绝对路径（`# /OI/数学` → 从根目录开始）和相对路径（`# 数学` → 在当前目录下）
- 自动匹配已有题单（同名则追加，新建则创建）
- 如果没有标题层级，整篇作为纯文本列表导入

### 导出

- 在左侧树形目录中，右键题单 → 「导出 Markdown」
- 导出为 `.md` 文件，保留 Part 和题目结构

## 随机跳题

从当前上下文的**未完成题目**中随机选择一道，并高亮显示：
- 题单内：「随机跳题」按钮
- 文件夹内：「随机跳题」按钮
- 全局：工具栏统计信息旁的「随机跳题」按钮

## 全局搜索

- 顶部搜索框，输入关键词实时搜索
- 匹配文件夹名、题单名、Part 名、题目名
- 结果按类型分组，匹配文字高亮显示
- 点击结果跳转到对应位置并高亮该题目

## 主题切换

- 工具栏右侧的暗黑/明亮切换开关
- 切换后立即生效，偏好自动保存

## 滚动位置记忆

- 每个题单/文件夹独立保存滚动位置
- 切换页面后自动恢复到上次离开的位置（瞬间跳转，无动画）

## 统计信息

各级页面顶部显示完成率统计：
- 全局：「已完成 N/M (X%)」
- 文件夹：递归统计所有子孙题单
- 题单：统计所有 Part 和直接题目
- Part：统计内部题目

## 开发

```bash
npm install        # 安装依赖
npm run dev        # 开发模式
npm run build      # 构建
npm run pack       # 打包分发 (electron-builder)
```

打包产物在 `release/` 目录：

| 文件/目录 | 说明 |
|---|---|
| `win-unpacked/` | 便携版目录，解压即用 |
| `TODO列表 Setup *.exe` | NSIS 安装包 |
| `TODO列表 *.exe` | 单文件便携版 |
